# Editorial Workflow

This workflow keeps content consistent, high-quality, and fast to ship.

## 1) Intake and Research
- Gather signals from GitHub, arXiv, and community channels.
- Capture research notes in the daily brief.
- Validate novelty and relevance against the query backlog.

## 2) Content Brief
- Use the content brief template for each new piece.
- Define audience, intent, and outcome.
- Identify needed visuals and code examples.

## 3) Draft
- Draft using the appropriate template (concept, guide, compare).
- Lead with a direct answer in the first paragraph.
- Add a human summary and 1-2 visuals or diagrams.

## 4) Review and QA
- Run the review rubric.
- Validate code examples and links.
- Ensure JSON-LD and metadata are in place.

## 5) Publish
- Commit content files.
- Update internal links and sitemap (if applicable).
- Post announcement to community channels.

## 6) Refresh Loop
- Identify refresh candidates weekly.
- Update timestamps and add new data points.
- Merge related content and deprecate duplicates.
