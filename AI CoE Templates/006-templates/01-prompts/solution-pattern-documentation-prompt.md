# Solution Pattern Documentation Prompt

**Use Case**: Create comprehensive design pattern documentation for the pattern library

## Prompt Structure:
```
Using the design-pattern-template.md, create a complete Design Pattern document for:

**Pattern Definition:**
- Pattern Name: [Descriptive pattern name]
- Pattern Category: [e.g., Data Analytics, Integration, Security]
- Primary Use Case: [Main problem this pattern solves]
- Industry Applications: [Industries where this pattern is commonly used]

**Technical Scope:**
- Core Technologies: [Primary OCI services and technologies]
- Architecture Style: [e.g., Microservices, Event-driven, Serverless]
- Data Patterns: [Data flow and processing patterns]
- Integration Approach: [How systems connect and communicate]

**Business Context:**
- Value Proposition: [Primary business benefits]
- ROI Drivers: [Key factors that drive return on investment]
- Typical Metrics: [Common success measurements]
- Implementation Complexity: [Complexity level and typical duration]

**Reference Information:**
- Similar Patterns: [Related or alternative patterns]
- Prerequisites: [Required capabilities or prior implementations]
- Scaling Considerations: [How pattern scales with growth]
- Common Variations: [Different ways to implement the pattern]

Generate a comprehensive Design Pattern document that:
1. Can be used by solution architects for pattern selection
2. Provides complete technical implementation guidance
3. Includes discovery questions for customer engagement
4. Contains detailed bill of materials for cost estimation
5. Serves as authoritative pattern documentation
6. Can be referenced across multiple customer engagements
```

## Example Usage Scenario

**Scenario**: Solution Pattern Documentation  
A solution architect needs to document a new real-time fraud detection pattern for the pattern library.

**Application**: Use this prompt with financial services context, real-time analytics, and fraud detection specifics to create comprehensive pattern documentation.
