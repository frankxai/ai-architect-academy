Evolved Platform Architecture

Experience Layer

  • Unified design system with adaptive theming and accessibility baked in.
  • Dynamic surfaces: onboarding wizard, journey canvas, retention arena, leaderboards, professor console.
  • Component-driven build (Storybook) to enable iterative delivery and future white-label offerings.

Intelligence Layer

  • Knowledge graph + vector store powering personalization, recommendations, and dependency navigation.
  • Orchestration engine blending heuristics, LLM reasoning, and reinforcement from learner feedback.
  • Agent services for ingestion, summarization, and retention question generation.

Platform & Ops Layer

  • Serverless core on Vercel with modular APIs (Next.js Route Handlers + Edge Functions).
  • Data backbone via managed Postgres/Supabase + graph DB (Neo4j Aura) and streaming event bus.
  • Observability mesh (PostHog, Langfuse, Datadog) integrated with ops runbooks.

Design System Upgrade

  • Create token library (spacing, typography, color ramps) aligned with accessibility contrast ratios.
  • Build core components: `KPCard`, `JourneyTimeline`, `GamificationMeter`, `RetentionModal`.
  • Adopt motion guidelines (micro-interactions for streak gains, dependency reveals).

Data Flows & Knowledge Fabric

Content Lifecycle

  • Agent Crawlers ingest new AI updates ? embed + classify ? submit to review queue.
  • Professor console approves, enriches metadata, assigns point/freshness multipliers.
  • Graph sync service reconciles hierarchy links and pushes notifications to impacted learners.

Learner Feedback Loop

  • Each KP completion logs rating, confidence, and reflection snippet.
  • Reinforcement job tunes recommendation weights and updates mastery map.
  • Insights dashboard surfaces trending gaps for content team action.

Analytics Pipeline

  • Event bus streams to warehouse (BigQuery/Snowflake) for cohort analytics.
  • Pre-computed KPI tiles feed ops dashboard and executive scorecards.
  • Anomaly detection flags drops in progress or engagement for proactive outreach.

Step-by-Step MVP Build Sequence

Step 1 · Vision Alignment (Day 0)

  • Lock personas, value map, and success metrics using service blueprint.
  • Finalize domain taxonomy and certification outcomes with professor council.

Step 2 · Platform Foundations (Day 0-1)

  • Bootstrap Next.js repo with design system tokens and Storybook.
  • Provision Supabase, Neo4j Aura, PostHog, Langfuse, and event bus.
  • Configure CI/CD, lint/test pipelines, preview deployments.

Step 3 · Onboarding Vertical Slice (Day 1-2)

  • Build onboarding wizard UI, connect profile API, persist learner record.
  • Implement persona classifier stub returning recommended track preview.
  • Ship Playwright smoke test for onboarding completion.

Step 4 · Knowledge Graph Core (Day 2-3)

  • Import initial hierarchy (domains ? knowledgepoints) and seed dependencies.
  • Create graph query API for playlist generation and dependency lookups.
  • Expose admin console stub to inspect relationships.

Step 5 · Journey Canvas & Playlist Logic (Day 3-4)

  • Render journey canvas with KP cards, durations, and point values.
  • Wire recommendation engine (LLM-assisted) with heuristics fallback.
  • Enable add/swap interactions respecting prerequisites.

Step 6 · Gamification Spine (Day 4-5)

  • Implement points service, streak tracker, certification progress model.
  • Expose dashboard widgets: streak meter, level badge, leaderboard preview.
  • Log events to analytics pipeline; validate reward calculations.

Step 7 · Retention Challenge Loop (Day 5-6)

  • Build retention pulse UI and API to request adaptive questions.
  • Integrate LLM generator using KP summaries + user mastery state.
  • Update mastery map and gamification points post-challenge.

Step 8 · Agent Ingestion & Ops (Day 6)

  • Configure crawler agent templates for LLM/industry news sources.
  • Deliver approval workflow in professor console with audit logging.
  • Test notification fan-out to learners impacted by new KPs.

Step 9 · Hardening & Insights (Day 6-7)

  • Run load tests, resolve performance hotspots, optimize graph queries.
  • Finalize dashboards (activation, engagement, outcomes) and alert thresholds.
  • Execute launch checklist dry run; capture open issues.

Step 10 · Launch & Iterate (Day 7)

  • Deploy MVP, onboard pilot cohort, monitor live metrics.
  • Host feedback session, log improvements into Sprint 1 backlog.
  • Celebrate wins; prep roadmap for professor program and builder mode.
