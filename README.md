## Quick Start

<a name="cross-platform"></a>

### Claude Code (Primary)

```bash
# Install the extension
git clone https://github.com/frankxai/ai-architect-academy.git
cd ai-architect-academy
./install.sh claude-code

# Open Claude Code
claude

# Start learning
/academy
```

### Cline / Cursor / Windsurf

```bash
# Same repo, different target
git clone https://github.com/frankxai/ai-architect-academy.git
cd ai-architect-academy

# Install for your agent
./install.sh cline       # Cline (VS Code)
./install.sh cursor      # Cursor
./install.sh windsurf    # Windsurf
```

## The Experience

When you launch your coding agent in this directory, you'll see:

```
Academy | 0/3 labs completed | Current: none | Type /academy for menu
```

Type `/start-lab 01` and the instructor begins:

```
╔══════════════════════════════════════════════════════════════╗
║  AI ARCHITECT ACADEMY — Lab Mode                            ║
║  Lab: Fix the Broken RAG Pipeline                           ║
║  Difficulty: Intermediate | Est. Time: 45m                  ║
╚══════════════════════════════════════════════════════════════╝

Welcome, Architect.

Your company's customer support AI is returning irrelevant answers.
The RAG pipeline is live but broken. There are 3 bugs in rag_pipeline.py.

Before I guide you — take a look at the code. What do you notice about
the chunk_documents() method?
```

The instructor uses **Socratic method** — questions, not answers. When stuck, type `/hint` for incremental clues. When ready, type `/review` for an architect-grade code review.

## How It Works (Architecture)

```
ai-architect-academy/
├── install.sh                      # Cross-platform extension installer
├── manifest.json                   # Extension metadata & skill registry
├── CLAUDE.md                       # Instructor Engine (Claude Code)
│
├── platforms/                      # Platform-specific adapters
│   ├── claude-code/                # .claude/ config, hooks, commands
│   ├── cline/                      # .clinerules, .cline/skills/
│   ├── cursor/                     # .cursorrules
│   └── windsurf/                   # .windsurfrules
│
├── labs/                           # Interactive coding labs
│   ├── 01-rag-pipeline/            # Python — fix broken RAG (3 bugs)
│   ├── 02-multi-agent-system/      # Python — build coordinator pattern
│   └── 03-mcp-server/              # TypeScript — build MCP server
│
├── skills/                         # Cross-platform skill packs
│   ├── core/                       # RAG, MCP, agents, architecture
│   ├── oci/                        # Oracle Cloud specialist skills
│   ├── multi-cloud/                # AWS, Azure, GCP, multi-cloud
│   └── enterprise/                 # Security, compliance, governance
│
├── dashboard/                      # Progress visualization
│   ├── index.html                  # Localhost dashboard (port 3721)
│   ├── styles.css                  # Dark theme, glassmorphic UI
│   └── app.js                      # Progress tracking engine
│
├── knowledge-base/                 # Reference documentation
│   ├── architecture-patterns/      # 20+ design patterns
│   ├── cloud-services/             # OCI, AWS, Azure, GCP guides
│   └── best-practices/             # Production deployment guides
│
├── assets/diagrams/                # 6 SVG architecture diagrams
├── diagrams/                       # D2 diagram source code
└── .academy/progress.json          # Your progress (gitignored)
```

### The Three Layers

1. **Instructor Engine** (`CLAUDE.md` / platform adapters) — Defines the teaching persona, Socratic rules, skill activation, and progress tracking. This is what turns a generic coding agent into a domain-expert instructor.

2. **Interactive Labs** (`labs/`) — Real codebases with real bugs and real test suites. You write code. The instructor guides. Tests are the judge.

3. **Knowledge Base** (80+ skills) — Deep-dive skills that auto-activate based on context. Working on RAG? The RAG skill loads. Building an MCP server? The MCP patterns load. Deploying to OCI? The cloud specialist activates.
