## Recommended Management Structure

### Repository Structure:
```
ai-architect-academy/           # Main OSS repo
├── dashboard/                  # SaaS app (separate deploy)
├── docs/                       # Public documentation
├── patterns/                   # All patterns (MIT)
└── .internal/                  # .gitignore'd internal docs

ai-architect-academy-internal/  # Private repo (optional)
├── strategy/                   # Business plans
├── financials/                 # Projections
└── partnerships/               # Partner agreements
```

### Deployment Strategy:
1. **GitHub** - OSS patterns, templates, workflows
2. **Vercel** - Dashboard app (from /dashboard folder)
3. **Notion/Linear** - Internal docs, task management

### Workflow:
```
OSS Contributors → GitHub PR → Review → Merge → Auto-deploy docs
SaaS Development → Dashboard PR → Review → Merge → Vercel deploy
Internal Planning → Notion/Linear → Private access
```
